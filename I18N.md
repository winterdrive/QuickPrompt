## ğŸ”§ How to Set Language

### Method 1: VS Code Global Locale Setting

1. Open VS Code settings (`Cmd+,` or `Ctrl+,`)
2. Search for "locale"
3. Find the "Locale" setting
4. Choose your desired language:
   - `en` - English
   - `zh-tw` - Traditional Chinese
   - `zh-cn` - Simplified Chinese
5. Restart VS Code

### Method 2: Via Command Palette

1. Open the Command Palette (`Cmd+Shift+P` or `Ctrl+Shift+P`)
2. Type "Configure Display Language"
3. Select your desired language
4. Restart VS Code

### Method 3: Edit settings.json

Add the following to your VS Code settings file:

```json
{
    "locale": "zh-tw"  // or "en", "zh-cn"
}
```

## ğŸ› ï¸ Developer Guide

### Add a New Language

1. **Create language files:**

   ```bash
   # Create package.json localization
   cp package.nls.json package.nls.[language-code].json
   
   # Create runtime messages
   cp i18n/en.json i18n/[language-code].json
   ```

2. **Translate content:**
   - Translate all strings in `package.nls.[language-code].json`
   - Translate all strings in `i18n/[language-code].json`
   - Keep placeholder syntax intact (e.g., `{0}`, `{1}`)

3. **Test the language:**
   - Set the corresponding locale in VS Code
   - Reload the extension
   - Verify all strings appear correctly

### Using i18n Strings in Code

Use the `I18n.getMessage()` method in your code:

```typescript
import { I18n } from './i18n';

// Basic usage
const message = I18n.getMessage('message.copied', promptTitle);

// With multiple parameters
const formatted = I18n.getMessage('status.lastUsed', timeText);

// In UI components
vscode.window.showInformationMessage(I18n.getMessage('message.promptAdded', title));
```

### String Key Naming Convention

- Use dot-separated namespaces: `category.subcategory.key`
- Categories include:
  - `message.*` - User messages (info, warning, error)
  - `input.*` - Input dialog prompts and placeholders
  - `confirm.*` - Confirmation dialog messages
  - `icon.*` - Icon-related strings
  - `status.*` - Status bar and tooltip messages
  - `time.*` - Time-related text

### Parameter Formatting

Strings can include numbered placeholders for dynamic content:

```json
{
    "message.copied": "âœ… Copied '{0}' to clipboard",
    "time.daysAgo": "{0} days ago"
}
```

In code:

```typescript
I18n.getMessage('message.copied', promptTitle);
I18n.getMessage('time.daysAgo', '5');
```

## ğŸ” Troubleshooting

### Q: The extension does not display the correct language

**Solution:**

1. Make sure VS Code's locale setting is correct
2. Reload the VS Code window
3. Check if the corresponding language file exists in `i18n/` directory
4. Verify `I18n.initialize()` is called in `activate()`

### Q: Some text still appears in English

**Possible reasons:**

1. The string is not yet internationalized
2. The language file is missing the translation
3. The code uses hardcoded strings

**Solution:**

1. Check if `i18n/[language].json` contains the string key
2. Make sure the code uses `I18n.getMessage()` instead of hardcoded text
3. Verify the key name matches exactly

### Q: How to revert to English interface

Set VS Code locale to `en` and reload.

### Q: Placeholders not working

Make sure you're using the correct format:

- In JSON: `{0}`, `{1}`, `{2}`, etc.
- In code: Pass arguments in order to `I18n.getMessage()`
